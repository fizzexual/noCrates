# noCrates 2.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans (inline) to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild noCrates as a PhoenixCrates-class crates plugin — 3-phase animations + idle particle shapes, first-class linked keys, deep rewards (limits/alternative/selective), guaranteed-win, rerolls, editor, migrations, and a real module/addon system with all PhoenixCrates add-on equivalents built in — one jar for MC 1.20 → 26.2.

**Architecture:** Kernel + modules. The kernel owns config/text/menu/action/storage/compat and the crate-key-reward-open-animation engine behind small SPIs (`PreOpenAnimation`, `PostOpenAnimation`, `RewardDisplayAnimation`, `IdleShape`, `ActionType`, `DataStore`, `Importer`, `CustomItemProvider`). Built-in modules (crate-claim, mass-opening, lootboxes, rarities, last-winner, animations-plus, chest-hunt) consume only the public `NoCratesApi`; the same `ModuleManager` loads external addon jars (`addons/*.jar` + `addon.yml`).

**Tech Stack:** Java 17 bytecode (JDK 21, `--release 17`), Maven + Shade, `io.papermc.paper:paper-api:1.20.1-R0.1-SNAPSHOT` (provided), Adventure/MiniMessage (server-bundled), bStats 3.1.0 + HikariCP 5.1.0 + mariadb-java-client 3.3.3 (shaded, relocated), JUnit 5.

## Global Constraints

- Compile: `maven.compiler.release=17`; build against paper-api `1.20.1-R0.1-SNAPSHOT` only. **No NMS, no packets, no InventoryView references, no version-specific imports.**
- Runtime: Paper/Purpur/Folia **1.20 → 26.2** (Java 17/21/25). `api-version: '1.20'`, `folia-supported: true`.
- Every `Material`/`Sound`/`Particle` by-name lookup goes through `compat.Compat` (never bare `valueOf` on renamed constants; rename chains for 1.20.5 changes, e.g. DUST↔REDSTONE, HAPPY_VILLAGER↔VILLAGER_HAPPY, ENTITY_EFFECT↔SPELL_MOB, POOF↔EXPLOSION_NORMAL, EXPLOSION↔EXPLOSION_LARGE, WITCH↔SPELL_WITCH, SMOKE↔SMOKE_NORMAL, LARGE_SMOKE↔SMOKE_LARGE, FIREWORK↔FIREWORKS_SPARK, ENCHANT↔ENCHANTMENT_TABLE, EFFECT↔SPELL, INSTANT_EFFECT↔SPELL_INSTANT, ITEM↔ITEM_CRACK, BLOCK↔BLOCK_CRACK, RAIN↔WATER_DROP, SPLASH↔WATER_SPLASH, BUBBLE↔WATER_BUBBLE, TOTEM_OF_UNDYING↔TOTEM, GUST_EMITTER_LARGE, DUST_COLOR_TRANSITION unchanged).
- All scheduling through `compat.Scheduling` (Folia region/entity/async when detected; BukkitScheduler otherwise). No bare `Bukkit.getScheduler()` outside `Scheduling`.
- Adventure: only stable surface (`Component.text`, `MiniMessage#deserialize`, `Audience#sendMessage/showTitle/playSound`, `LegacyComponentSerializer`). Nothing deprecated in adventure 4.x (removed in 5.x on Paper 26.2).
- All player-facing text via `text.Lang` keys from `languages/<code>.yml`. MiniMessage; `&`-legacy and `&#RRGGBB` inputs auto-converted.
- Base package `com.nocrates`. Main class `com.nocrates.NoCratesPlugin`. Plugin name `noCrates`, version 2.0.0.
- All hooks/soft-deps reflection-guarded; plugin fully functional with none installed.
- **Build gate at each phase end:** `mvn -q clean package` green (tests + shaded jar) before commit/push.
- Commits: conventional style, one per task cluster.

```java
// compat
public final class Compat {
    public static Material material(String name, Material fallback);
    public static Particle particle(String name);          // null if unresolvable
    public static Sound sound(String name);                // null if unresolvable
    public static void play(Player p, String sound, float vol, float pitch);
    public static void spawnDust(Location at, org.bukkit.Color color, float size, int count,
                                 double ox, double oy, double oz);
    public static void spawn(Particle part, Location at, int count,
                             double ox, double oy, double oz, double speed);
}

    public static void later(Plugin pl, Location ctx, long ticks, Runnable r);
    public static Cancellable timer(Plugin pl, Location ctx, long delay, long period, Runnable r);
    public static void entity(Plugin pl, org.bukkit.entity.Entity e, Runnable r);
    public static void async(Plugin pl, Runnable r);
    public interface Cancellable { void cancel(); }
}

    public Component raw(String key, TagResolver... tags);

    Lang lang(); DataStore storage(); StatsService stats();
    void registerPre(PreOpenAnimation a); void registerPost(PostOpenAnimation a);

    void registerAction(ActionType t); void registerImporter(Importer i);
    void registerPlaceholder(String sub, java.util.function.BiFunction<org.bukkit.OfflinePlayer,String,String> fn);
}

// migrate
public interface Importer { String id(); boolean detect(); ImportReport run(); }
```
