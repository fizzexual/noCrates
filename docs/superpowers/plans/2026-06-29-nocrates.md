# noCrates Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans (inline) to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build noCrates — an advanced, free Minecraft crates plugin with an in-game GUI editor, every common crate feature, plus a right-click Lootbox and a novel pick-N-of-M Chest Hunt — as a single jar spanning MC 1.20→26.1.2.

**Architecture:** Modular monolith (Maven, one shaded jar). Roll-then-animate-then-grant flow: the open controller rolls rewards up front, hands a `CrateSession` to a pluggable `Animation` for visuals only, then grants on completion. Version drift is absorbed by a `VersionCompat` layer (config-driven material/sound/particle keys); no NMS.

**Tech Stack:** Java 17 bytecode (javac 21 `--release 17`), Paper API 1.20.1, Maven Shade, Adventure/MiniMessage, HikariCP (MySQL), bStats, JUnit 5 + MockBukkit.

## Global Constraints

- **Compile:** `maven.compiler.release = 17`. Build against `io.papermc.paper:paper-api:1.20.1-R0.1-SNAPSHOT`.
- **No NMS / no version-specific imports.** Only stable Bukkit/Paper API.
- **Runtime range:** must load on MC 1.20 (Java 17) → 26.1.2 (Java 25). Resolve `Material`/`Sound`/`Particle` via `VersionCompat`, never hard-fail on a missing constant.
- **Group/base package:** `com.nocrates`. Plugin main: `com.nocrates.NoCrates`.
- **Player-facing text:** all via `messages.yml` + MiniMessage. No hardcoded chat strings.
- **All hooks soft-depend.** Plugin must enable fully with zero optional deps present.
- **Build gate:** every phase ends with a clean `mvn -q package` producing a valid shaded jar before commit/push.
- **Commits:** frequent, conventional-commit style. Push to `github.com/fizzexual/noCrates`.

## Core Interfaces (contracts — keep signatures stable across tasks)

```java
// animation/Animation.java
public interface Animation {
    String id();                       // "instant", "csgo", ...
    void play(CrateSession session);   // visuals only; calls session.finish() when done
}

// animation/CrateSession.java
public final class CrateSession {
    public Player player();
    public Crate crate();
    public java.util.List<Reward> outcome(); // pre-rolled rewards to reveal/grant
    public void finish();                    // grants outcome + records, idempotent
}

// reward/RewardAction.java
public interface RewardAction { void execute(Player player); }
// reward/RewardActions.java
public final class RewardActions {
    public static RewardAction parse(String line); // "item: DIAMOND 5" -> action
}

// reward/RollEngine.java
public final class RollEngine {
    public RollEngine(java.util.Random rng);
    public Reward roll(java.util.List<Reward> rewards); // weighted by Reward.weight()
}

// storage/DataStore.java
public interface DataStore {
    java.util.concurrent.CompletableFuture<PlayerData> load(java.util.UUID id);
    void save(PlayerData data);
    void close();
}

// storage/PlayerData.java key surface
public final class PlayerData {
    public int keys(String crate, String keyType);
    public void addKeys(String crate, String keyType, int amount);
    public int opens(String crate);
    public void incrOpens(String crate);
    public int winCount(String crate, String rewardId);
    public void incrWin(String crate, String rewardId);
}
```

- [ ] **Failing test:** seeded RNG, rewards weights [50,2]; over 100k rolls the ~50:2 ratio holds within tolerance; zero-weight never rolled; single reward always returned.

```java
@Test void distributionMatchesWeights() {
  var r = new RollEngine(new Random(42));
  var a = reward("a", 50), b = reward("b", 2);
  int ca = 0, n = 100_000;
  for (int i=0;i<n;i++) if (r.roll(List.of(a,b)).id().equals("a")) ca++;
  double share = ca / (double) n;          // expect ~50/52 = 0.962
  assertTrue(Math.abs(share - 50.0/52.0) < 0.02);
}
```

- [ ] Run → fail; implement cumulative-weight roll; run → pass. Commit `feat(reward): weighted roll engine`.
